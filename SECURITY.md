## 1. Introduction
This document outlines the security policy for the **Intelligent Academic Problem Solving System (IAPSS)**.  
The purpose of this policy is to define acceptable use, responsible disclosure practices, and security-related boundaries for users interacting with the source code and the deployed application.

This project is developed and deployed primarily for **academic, learning, and evaluation purposes**.

---

## 2. Supported Versions
Only the **latest deployed version** of the application and the **main branch** of the repository are considered active and supported.

- Experimental branches
- Older commits
- Unreleased features

are not guaranteed to receive security updates or fixes.

---

## 3. Intended Use
The application and its source code are intended to be used for:
- Academic demonstration
- Learning and understanding system architecture
- Project evaluation by authorised faculty or institutions

The system is **not intended for production or commercial use**.

---

## 4. Prohibited Activities
Without prior written permission from the author, the following activities are **strictly prohibited**:

- Attempting to gain unauthorised access to backend services or databases
- Exploiting API endpoints or hidden routes
- Performing penetration testing, vulnerability scanning, or automated attacks
- Brute-force attempts on authentication or input fields
- Reverse engineering application logic for misuse
- Deliberate performance degradation or denial-of-service attempts
- Extraction, modification, or manipulation of stored or transmitted data

Any such activity may be treated as **unauthorised use or misuse** of the system.

---

## 5. Responsible Disclosure
If you discover a potential security vulnerability or unintended behaviour:

- Do **not** exploit the issue
- Do **not** publicly disclose the issue
- Do **not** attempt repeated testing

Instead, report the issue responsibly using one of the following methods:
- Open a GitHub Issue with a high-level description (avoid sensitive details)
- Contact the author through the GitHub profile

Please include:
- A clear description of the issue
- Steps to reproduce (if applicable)
- Expected vs actual behaviour

Responsible disclosure is appreciated and encouraged.

---

## 6. Security Measures and Limitations
While reasonable security practices are followed during development:
- The application does not claim to be fully secure
- No guarantee is made against all forms of attack
- Security mechanisms may be basic due to academic scope

Users should interact with the system **ethically and responsibly**.

## 7. Data and Privacy
